# ADR-001: 技术栈选型

## 状态

已采纳 (2026-06-21)

## 上下文

图片格式转换器是一个纯前端 SPA，需要：
- 组件化 UI 框架
- 快速构建工具
- 响应式样式方案
- Canvas API 图片转换能力
- 浏览器自动化测试
- 公网子路径部署

## 决策

| 选项 | 决定 | 理由 |
|---|---|---|
| 框架 | **React 18 + TypeScript** | 生态最成熟、社区最大、TypeScript 集成度最高。备选 Vue 3 同样优秀，但 React 在招聘和社区资源方面略胜一筹。 |
| 构建 | **Vite 5** | 开发 HMR 极速、零配置 TypeScript、base path 配置简单。相比 webpack 显著更快，相比 CRA 更现代化。 |
| 样式 | **Tailwind CSS 3** | 原子化 CSS 减少样式文件数量、内置响应式断点、JIT 编译体积小。备选原生 CSS 也能完成，但开发效率低。 |
| 状态管理 | **React Context + useReducer** | 状态结构简单（模式、文件列表、结果列表），无需引入 Redux/Zustand 等第三方库。Context 足够且不增加依赖。 |
| 格式转换 | **HTML5 Canvas API** | 浏览器原生、无依赖、PNG/JPEG/WebP 原生支持。BMP 输出需自研编码器。 |
| 批量打包 | **JSZip** | 纯 JS、浏览器兼容好、支持 Blob 生成、社区活跃。 |
| 测试 | **Playwright** | 真实浏览器环境、多浏览器支持、截图对比、支持文件拖拽/下载测试。备选 Cypress 不支持多标签页。 |

## 不引入的库及理由

- **前端路由库**（react-router）：单页应用无需路由
- **Redux/Zustand**：状态结构简单，Context 足够
- **axios**：无 API 调用
- **UI 组件库**（Ant Design / Material UI）：增加包体积，Tailwind 手写组件更轻量

## 影响

- 开发只需 Node.js + npm，无额外环境依赖
- 产物为纯静态文件（HTML + JS + CSS），Nginx 即可部署
- TypeScript 提供类型安全保障
- 总依赖数约 5-8 个（react, react-dom, typescript, vite, tailwindcss, jszip, playwright 测试）

## 备选方案

### 方案 B：Vue 3 + Vite + Tailwind CSS
- Vue 3 Composition API 与 React Hooks 功能对等
- 同样拥有 Vite 原生支持和 TypeScript 集成
- 国内市场占有率高于 React
- **未采纳原因**：React 全球生态更大，库和教程资源更丰富；技术要求文档中"React 或 Vue"顺序 React 在前

### 方案 C：原生 JS + Vite + 原生 CSS
- 最小化依赖，极简打包体积
- 但缺乏组件化，UI 状态管理困难
- **未采纳原因**：缩放性和可维护性不足，且项目约束已要求 React/Vue
